# Business Rules Maintainer

Use this skill when maintaining the company Business Rules, answering questions from it, ingesting durable source material, refreshing the index, or linting brms structure.

Before changing brms files, resolve the configured brms root, read its AGENTS.md, inspect brms/index.md and recent brms/log.md entries, then use the Business Rules plugin tools for source reads, rule writes, patch proposals, backlinks, and logging.

Keep raw sources immutable, cite brms rules and raw paths, update brms/index.md when rule navigation changes, and append a concise brms/log.md entry for durable updates. For Paperclip project work, keep `brms/projects/<project-slug>/standup.md` current as the executive status view and use `brms/projects/<project-slug>/index.md` for durable project knowledge. Write project material as concept-grouped executive synthesis, not issue-id lists or metadata dumps.
